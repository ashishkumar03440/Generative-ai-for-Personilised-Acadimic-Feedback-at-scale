# 🖥 FrontEnd — EduAI React Web Application

> **Technology:** React 18 + Vite + TypeScript + Tailwind CSS + shadcn/ui  
> **Entry Point:** `index.html` → `src/main.tsx`  
> **Dev Server:** `npm run dev` (Vite — default port 5173)

---

## 👥 User Roles & Routing

The app serves three distinct user roles, each with their own set of pages:

| Role | Login Redirect | Pages Accessible |
|------|---------------|-----------------|
| `student` | `/student/dashboard` | Dashboard, Assignments, Submit, Feedback, Progress |
| `teacher` | `/teacher/dashboard` | Dashboard, Create Assignment, Review Editor, Analytics |
| `admin` | `/admin/dashboard` | Dashboard, User Management, AI Monitor, Audit Log |

Role-based routing is enforced in `src/App.tsx` using the `AuthContext`.

## 🎨 Design System

| Token | Choice |
|-------|--------|
| CSS Framework | **Tailwind CSS** with custom design tokens in `index.css` |
| Component Library | **shadcn/ui** (Radix UI primitives + Tailwind styling) |
| Theme | **Dark/Light mode** switchable via `ThemeContext` |
| Typography | Inter, Outfit (Google Fonts) |
| Charts | **Recharts** (via shadcn/ui chart components) |

CSS variables for the design system are defined in `src/index.css` and toggled via the `dark` class applied to `<html>`.

---

## 🔐 Authentication

- `AuthContext` (in `src/contexts/AuthContext.tsx`) holds the logged-in user info and JWT access token in memory (React state)
- The access token is sent on every API request as `Authorization: Bearer <token>`
- The refresh token is stored in an **HttpOnly cookie** (not accessible to JS)
- On app load, `/user/me` is called to restore session from a valid cookie
